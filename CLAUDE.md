# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

clash-for-linux is a Bash-based automation toolkit for deploying and managing Clash/Mihomo proxy on Linux servers. It provides systemd service integration, multi-subscription management, and a pre-configured Dashboard (MetaCubeXD).

## Common Commands

```bash
# Installation
sudo make install

# Service management (via clashctl or make)
make start
make stop
make restart
make status
make update              # Refresh subscription

# Or using clashctl
clashctl start
clashctl stop
clashctl restart
clashctl status
clashctl update

# Multi-subscription management
clashctl sub add <name> <url> [headers]
clashctl sub del <name>
clashctl sub use <name>
clashctl sub update [name]
clashctl sub list

# System proxy helpers (after source /etc/profile.d/clash-for-linux.sh)
proxy_on
proxy_down

# Uninstall
sudo make uninstall
```

### Key Files

| File | Purpose |
|------|---------|
| `.env` | Environment variables (CLASH_URL, CLASH_SECRET, ports) |
| `conf/config.yaml` | Clash runtime configuration |
| `conf/subscriptions.list` | Multi-subscription storage (`name\|url\|headers\|updated`) |
| `conf/mixin.d/*.yaml` | Configuration overlays (merged in alphabetical order) |
| `conf/Country.mmdb` | GeoIP database for GEOIP rules |

### Startup Flow

1. `scripts/cmd/start-service.sh` loads `.env` and detects CPU architecture
2. Downloads subscription from `CLASH_URL`
3. Converts non-Clash YAML via subconverter (if needed)
4. Injects `external-controller`, `external-ui`, `secret` into config
5. Applies Mixin overlays from `conf/mixin.d/`
6. Creates symlink: `conf/ui -> dashboard/public/`
7. Starts Clash kernel from `libs/clash/`

### Configuration Hierarchy

```
Subscription config < Mixin configs < Runtime injection (controller/ui/secret)
```

### systemd Integration

- Service runs as `root` user
- `SYSTEMD_MODE=true` enables fallback behavior on subscription failures
- Unit file generated by `scripts/cmd/setup-systemd.sh`

## Script Conventions

### Script Naming

| Location | Naming Style | Example |
|----------|--------------|---------|
| `scripts/lib/` | 功能描述 | `get_cpu_arch.sh`, `resolve_clash.sh` |
| `scripts/cmd/` | 动作-对象 | `install-service.sh`, `start-service.sh` |
| Special | 约定俗成 | `shutdown.sh` |

### Server_Dir Calculation

All scripts calculate the project root directory relative to their location:

```bash
# For scripts in scripts/cmd/
Server_Dir="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"

# For scripts in scripts/lib/
Server_Dir="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
```

### Function Naming

| Prefix | Purpose |
|--------|---------|
| `ensure_` | Guarantee resource exists (`ensure_subscription_file`) |
| `resolve_` | Return computed value (`resolve_clash_bin`) |
| `force_` | Overwrite/update (`force_write_secret`) |
| `is_` | Boolean check (`is_running`) |

## Dashboard

- **Type**: MetaCubeXD (pre-compiled Nuxt.js static files)
- **Access**: `http://127.0.0.1:9090/ui`
- **Authentication**: Secret via `Authorization: Bearer <secret>` header
- **API**: Clash RESTful API at `/api/*`, WebSocket at `/logs`, `/traffic`, `/connections`
- **No separate web server needed** - Clash kernel serves static files

## Multi-Subscription Data Format

```
# conf/subscriptions.list
name|url|headers|updated
office|https://...|User-Agent: Clash|2025-01-15T10:00:00Z
personal|https://...|-|-
```

When `clashctl sub use <name>` is called, it updates `.env` with `CLASH_URL`, `CLASH_HEADERS`, and `CLASH_SUBSCRIPTION`.

## Security Defaults

- `external-controller` binds to `127.0.0.1:9090` only
- `secret` auto-generated if not specified (64-char hex)
- TLS verification enabled by default
- Recommend SSH port forwarding for Dashboard access

## Installation

- **Source = Install location**: The project directory is the installation directory
- No copying to `/opt` by default
- `clashctl` is symlinked to `/usr/local/bin/clashctl`
- Profile helpers: `/etc/profile.d/clash-for-linux.sh`

## Git Commit Guidelines

### Files NOT to commit (runtime generated)

| File | Reason |
|------|--------|
| `.env` | Contains sensitive data (subscription URL, secret) |
| `conf/config.yaml` | Runtime configuration, auto-generated from subscription |
| `conf/GeoSite.dat` | Geosite database, downloaded at runtime |
| `conf/cache.db` | Clash cache database |
| `conf/ui` | Symlink to dashboard, created at runtime |
| `libs/subconverter/pref.ini` | Subconverter config, generated at runtime |
| `libs/subconverter/subconverter` | Symlink to architecture-specific binary |
| `volumes/geoip/*.mmdb` | User's GeoIP database |
| `volumes/mixin.d/*.yaml` | User's custom mixin configs |
| `logs/*.log` | Log files |
| `temp/*` | Temporary files |

### Files to commit

| File | Reason |
|------|--------|
| `scripts/**/*.sh` | All shell scripts |
| `Makefile` | Build commands |
| `.gitignore` | Ignore rules |
| `README.md` | Documentation |
| `libs/clash/*` | Clash kernel binaries |
| `libs/subconverter/linux-*/*` | Subconverter binaries by architecture |
| `dashboard/public/*` | Dashboard static files |
| `conf/fallback_config.yaml` | Fallback configuration template |
